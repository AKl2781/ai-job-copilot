# Demo 演示脚本

## 演示目标

主演示控制在约 2 分钟，展示“招聘页 JD → Manifest V3 扩展 → 本地 FastAPI → DeepSeek → 结构化匹配结果”的真实闭环。需要回答工程细节时，再使用 4–5 分钟扩展讲解版。

## 演示前准备

- 在项目根目录准备未提交的 `.env`，确认 `LLM_API_KEY` 可用。
- 首次演示 Native Messaging 时，从 `edge://extensions` 复制真实扩展 ID，运行
  `install_native_host.bat <扩展ID>`，然后重新加载扩展。不要猜测 ID。
- 推荐打开岗位页面后按 `Alt+J`，直接点击“分析岗位”；扩展会在需要时自动启动本地服务。手动诊断时也可以双击 `start_ai_job_copilot.bat`，或运行：

  ```powershell
  uvicorn backend.app.main:app --reload --host 127.0.0.1 --port 8000
  ```

- 访问 <http://127.0.0.1:8000/health>，确认返回 `{"status":"ok"}`。
- 在 Edge 中以“加载解压缩的扩展”方式加载 `extension` 目录并固定图标。
- 预先选择一个公开、无需登录且页面结构稳定的招聘详情页；准备一段不含隐私信息的候选人资料。
- 关闭通知，放大浏览器到易读比例，不展示 `.env`、控制台请求头或真实 API Key。
- 演示前完整走一遍流程；保留截图作为网络或 API 临时不可用时的备份材料。

## 2 分钟快速演示

| 时间 | 操作 | 一句话讲解 |
| --- | --- | --- |
| 0:00–0:10 | 展示 popup 中已停止的本地服务 | “后端不需要开机常驻，分析时会按需启动。” |
| 0:10–0:20 | 打开公开招聘岗位页面 | “我从一个真实岗位详情页开始演示。” |
| 0:20–0:30 | 按 `Alt+J` 打开扩展 | “这是 Edge 优先、兼容 Chrome 的 Manifest V3 扩展。” |
| 0:30–0:45 | 展示自动提取的标题、URL 和 JD | “popup 打开后自动识别岗位详情，复杂页面也可以先选中 JD 再读取。” |
| 0:45–0:58 | 检查候选人资料 | “候选人资料可以编辑并保存在扩展 localStorage，分析只应依据这里的真实信息。” |
| 0:58–1:08 | 点击“分析岗位” | “扩展先确保本地 FastAPI 健康；未运行时自动启动，再发送一次分析请求。” |
| 1:08–1:45 | 展示 score 卡片、技能标签与评分依据折叠区 | “DeepSeek 提取要求和证据，后端按固定权重计算总分；matched、partial、missing、unverified 会分开展示。” |
| 1:45–2:00 | 编辑 greeting 并点击“复制打招呼文案” | “文案可以修改和复制，但项目不会自动发送消息；分数也只作求职辅助参考，不等于录用概率。” |

演示结束后双击 `stop_ai_job_copilot.bat` 停止后端。

结束时可点击“停止本地服务”。手动启停按钮是可选控制入口；正常流程只需点击一次“分析岗位”，无需确认框。
默认自动化验收不执行最后的真实 DeepSeek 分析请求。

## 4–5 分钟扩展讲解版

在完成上述 2 分钟流程后，按以下顺序补充约 2–3 分钟：

| 补充时间 | 内容 | 一句话讲解 |
| --- | --- | --- |
| 0:00–0:35 | 架构 | “扩展负责页面读取和交互，本地 FastAPI 负责模型调用、校验与异常处理。” |
| 0:35–1:00 | API Key 后端隔离 | “API Key 只从后端环境或 `.env` 读取，不写入扩展代码和分析请求。” |
| 1:00–1:40 | 智能提取逻辑 | “提取顺序是选中文字、候选区域评分，再回退到 main、article、role-main 和 body。” |
| 1:40–2:10 | 确定性评分 | “DeepSeek 只提取要求和候选人证据，后端使用固定状态映射与维度权重计算 score，并返回可展开的 score_breakdown。” |
| 2:10–2:40 | 当前限制 | “这是未经大规模用户验证的本地 MVP，没有数据库、云端部署、自动投递或自动聊天。” |

扩展讲解后的总时长应控制在 4–5 分钟。若现场时间有限，只保留快速版，不展开代码细节。

## 推荐重点

1. 先展示不选中文字的智能识别；需要说明兜底时，再展示选中文字优先。
2. 候选人资料应包含能力强弱层次，例如“掌握 Python”“了解 SQL”“Docker 基础学习阶段”，不要为了演示效果夸大经历。
3. 强调工程闭环：最小权限、Key 后端隔离、证据提取、确定性评分、结构化输出、loading/错误状态和自动化测试。

## 失败时的处理话术

- 页面无法读取：“浏览器对内置页或受限页禁止脚本注入；我切换到普通招聘详情页，或直接手动粘贴 JD。”
- 智能识别带入噪声：“这是通用启发式识别的边界。当前可以手动修剪，或通过选中文字获得更确定的输入。”
- DeepSeek 请求失败：“扩展和后端链路仍可用；上游调用可能受 Key、网络、限流或超时影响。后端已把这些异常转换成安全错误。”
- 输出不理想：“这是模型辅助分析，不保证结论正确；产品设计上保留了原始 JD 和 candidate_profile 供用户核验。”

不要在失败时声称“所有网站都支持”“模型结果完全准确”或“生产环境已稳定运行”。

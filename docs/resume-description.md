# 简历项目描述

下面提供不同长度的真实表述，可按版面选用。不要同时堆叠所有版本。项目当前是本地 MVP，没有数据库或云端部署，未进行大规模用户验证；模型评分仅供参考，也没有实现自动投递或自动聊天。

## 一句话版本

AI Job Copilot｜基于 Manifest V3、FastAPI 与 DeepSeek 的本地求职助手 MVP，实现招聘页 JD 提取、候选人资料匹配、严格 JSON 校验和结构化结果展示。

## 标准版本

**AI Job Copilot｜个人项目｜JavaScript / Manifest V3 / FastAPI / Pydantic / DeepSeek**

- 开发 Edge 优先、兼容 Chrome 的 Manifest V3 扩展，实现 popup 自动读取岗位、`Alt+J` 快捷唤起、选中文字优先及 `main/article/role-main/body` 多级回退。
- 设计通用岗位详情启发式提取，根据关键词、文本结构、交互占比与父子节点关系筛选正文，并清理部分按钮和尾部推荐职位噪声。
- 搭建 FastAPI 分析接口，将 `candidate_profile` 与 JD 分离传递给 DeepSeek；API Key 仅由后端环境读取，避免出现在扩展代码和请求载荷中。
- 使用严格 Pydantic 模型校验匹配度、技能差距、学习建议、判断理由和沟通文案等结构化结果，并处理配置、认证、限流、超时、连接和格式异常。
- 编写 pytest 与 Node 测试覆盖接口契约、候选人资料、本地保存、扩展权限和 6 类岗位提取场景；文档整理时 pytest 17 项及 Node 测试通过，并完成真实 DeepSeek 联调。

## 精简三点版本

- 使用 Manifest V3 开发 Edge/Chrome 扩展，实现招聘页 JD 自动提取、选中文字优先、智能正文识别和多级语义回退。
- 使用 FastAPI + DeepSeek 完成候选人资料与岗位要求的结构化分析，通过后端 Key 隔离和严格 Pydantic 校验稳定前后端契约。
- 以 pytest 和 Node 覆盖接口、异常、权限、localStorage 与提取场景；完成真实 API 联调，当前交付为本地 MVP。

## 面向不同岗位的侧重点

### 后端 / Python 岗

突出 FastAPI 请求模型、DeepSeek 服务封装、Pydantic 严格响应校验、安全错误映射和 pytest。不要把当前项目描述成微服务、分布式系统或生产级 API 平台。

### 前端 / 浏览器扩展岗

突出 Manifest V3、最小权限、脚本注入、DOM 内容提取、localStorage、异步请求与结构化渲染。不要声称做了复杂前端框架或跨浏览器自动化测试。

### AI 应用 / Agent 岗

突出模型输入边界、候选人资料真实性约束、结构化输出、严格校验、异常处理和真实 API 联调。当前是单次分析工作流，不应包装成自主 Agent、多智能体系统、RAG 或模型训练项目。
